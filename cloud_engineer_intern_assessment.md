## Objective

This assessment tests your understanding of cloud-native application development, containerisation, and infrastructure-as-code.  You will build a REST API, containerise it, and provision cloud infrastructure to host it.

## Problem Statement

CloudEngine Labs is onboarding a new micro-service that must be accessible over the internet from a cloud VM.  Your task is to build the service, containerise it, and automate the VM provisioning using Terraform so the whole stack can be reproduced from a single `terraform apply`.

## Assignment Tasks

### Task 1: Python Flask Application **[Required]**

Create `app.py` – a Flask application with one endpoint:
`GET /` → `{ "message": "Hello CloudEngine Labs - from Python Flask!" }`

Test locally with `flask run` or `python app.py` and verify the JSON response.

### Task 2: Containerise the Application **[Required]**

Write a `Dockerfile` that:
- Installs Flask
- Copies `app.py`
- Exposes port `5000`
- Starts the Flask app

Build the image and run it locally.  Push the image to Docker Hub or GitHub Container Registry and note the full image tag.

### Task 3: Terraform Infrastructure **[Required]**

Write Terraform code (`main.tf`, or split into `main.tf` / `variables.tf` / `outputs.tf`) to:
1. Launch a cloud VM (e.g. EC2 t2.micro on AWS with Ubuntu 22.04)
2. Configure a security group / firewall rule allowing port `5000` inbound
3. Use a `user_data` script (or equivalent) to install Docker, pull your image, and start the container on port `5000`

Run `terraform init && terraform plan && terraform apply` and capture the output.

### Task 4: Deploy & Validate **[Required]**

After `terraform apply` completes, access `http://<public-ip>:5000/` and confirm the API returns:
`{ "message": "Hello CloudEngine Labs - from Python Flask!" }`

Include the public URL (or screenshot) in your README.

### Task 5: GitHub Actions Pipeline (Bonus) **[Bonus]**

Create `.github/workflows/ci-cd.yml` that:
1. Builds and pushes the Docker image on every push to `main`
2. (Optional) Runs `terraform apply` to re-deploy the infrastructure
